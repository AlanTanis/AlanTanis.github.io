PHP 性能优化

发表于 2021-06-06 分类于 开发者 Valine：

PHP 项目性能优化，不要仅局限于优化 PHP。

PHP 性能问题解决方向：

  • PHP 语言级的性能优化（语法等基本功能）
  • PHP 周边问题的性能优化（周边服务，缓存，数据库等）
  • PHP 语言自身分析、优化

PHP 语言级性能优化

优化点 1：少写代码，多用 PHP 自身能力

PHP 代码需要编译为 C 语言再编译为机器语言，这一过程每次请求都会处理一遍，开销较大。

自写代码可能有较多冗余。因此，尽量使用 PHP 自身内置的变量、常量、函数等。

值得一提的是，目前很多的 PHP 缓存服务，都是缓存 opcode，从而提高性能。

优化点 2：PHP 内置函数的性能优劣

PHP 的内置函数之间，性能依然存在差异。了解 PHP 内置函数的时间复杂度是有必要的。

优化点 3：少用魔法函数

PHP 提供的魔法函数性能并不佳，尽可能避免使用。

优化点 4：错误抑制符 @

@ 符号的逻辑，是在该行代码的执行前后，分别增加 BEGIN_SILENCE 和 END_SILENCE 的 opcode，从而对 Zend 引擎的报错等级做了设置。所以，尽量别用 @ 符号。

优化点 5：合理使用内存

即使 PHP 有内存回收机制，但也要尽量利用 unset() 及时释放内存。

优化点 6：正则表达式

正则表达式的回溯开销较大。有时候，可以考虑使用字符串处理函数来实现。

优化点 7：避免在循环内做运算

一个简单的例子就是：

for 的每次循环，都会重新计算 $a+$b。因此，把运算逻辑，放到循环外部：

优化点 8：减少计算密集型业务：

PHP 不适合做大数据量的运算。PHP 更适合衔接 Web Server 与后端服务，呈现 UI，起到一个纽带的角色。

优化点 9：使用带引号字符串作为数组键值

PHP 会将没有带引号的键值作为常量，从而产生寻找常量的开销。这一点，一般情况下会有 notice。因此，严格使用引号，以字符串作为键值。

PHP 周边问题性能优化

PHP 周边包括：

  • 系统环境
  • 文件存储（硬盘）
  • 数据库
  • 缓存（MemCache，Redis 等）
  • 网络

优化的过程中，关注主要矛盾，找到问题的核心，才可能有较好的效果。

优化点 1：减少文件类操作

常见 PHP 场景的开销次序为：读写网络数据 > 读写磁盘 > 读写数据库 >> 读写内存。

因此，可以考虑减少网络数据和磁盘数据交互，尽量让 PHP 跟内存打交道。

优化点 2：优化网络请求

网络请求涉及 API 稳定性和网络稳定性等因素。因此可以：

  1. 设置超时时间
    a. 连接超时（约 200ms）
    b. 读超时（小于 1s）
    c. 写超时（约 500ms）

  2. 将串行请求并行化
    a. 使用 curl_multi_*()
    b.使用 swoole 扩展

  3. 压缩 PHP 的 API 输出

    当接口数据大于 100k 的时候，可以考虑使用 Gzip 压缩 API 输出，让客户端能更快地获取数据。

    代价是服务端和客户端都会产生额外的 CPU 开销。

优化点 3：缓存重复计算内容

对多次请求但内容相同的重复计算场景，设置缓存。例如，项目的模板渲染等。

优化点 4：重叠时间窗口思想

如果后续的任务不强依赖前面的任务，那么可以考虑重叠时间窗口。

优化点 5：旁路方案

旁路方案的实现，也要求后续的任务不强依赖前面的任务。

关于 BFC

根据 Mozilla 的 MDN 文档1中关于 BFC（Box Formatting Content，块级格式化上下文）的说明：

A block formatting context is a part of a visual CSS rendering of a web page. It’s the region in which the layout of block boxes occurs and in which floats interact with other elements.

…

Formatting contexts affect layout, but typically, we create a new block formatting context for the positioning and clearing floats rather than changing the layout, because an element that establishes a new block formatting context will:

  • contain internal floats.
  • exclude external floats.
  • suppress margin collapsing.

讲简单点，BFC 是页面上的一个独立容器，内部的元素不会影响外部的元素，反之亦然。

而盒子是否形成 BFC，对元素的 CSS 浮动有重要影响。同时，BFC 还能取消 margin 塌陷（margin collapsing），以及阻止元素被 float 的元素覆盖。

Surface Pro 7 DPST 设置

发表于 2020-02-09 分类于 数码 Valine：

又入手了一台 Surface Pro。然而 DPST（Display Power Saving Technology，显示器节能技术）对我而言，必须得关！

早前有一篇日志已经讨论过 Surface Pro 4 和 The New Surface Pro（5th Gen）关闭 DPST 的方法1。Surface Pro 7 的操作方法也类似，只不过项目在注册表中的位置和数值稍有变化。

Surface Pro 7 注册表中关于 DPST 的设置位于

定位到 FeatureTestControl 项，其默认值为 200。

之前的日志里有提到：

FeatureTestControl 的字段中，每个值都代表了一个功能特性。其中 0 表示启用某个特性，而 1 则表示禁用。Intel 的显卡驱动正是通过读取该注册表项中的值来决定是否启用相关特性。而 DPST 的值，就在从右往左数第五位。

十六进制 200 的转换成二进制为 0010 0000 0000，把从右往左数的第五位改成 1，即关闭 DPST，那么值为 0010 0001 0000，换算回十六进制为 210。

只要把 FeatureTestControl 的值改成 210 即可关闭 DPST。

修改完毕后，重启系统。

简单而言，fit_transform() 就是整合了 fit() 和 transform() 的功能。当我在训练集上用 fit_transform() 做 scale 的时候，它会先计算均值和方差，并记录下来，这就是 fit() 的功能。然后，再对数据执行归一化，也就是 transform()。
